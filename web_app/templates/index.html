围绕内核内存回收、Android Bugreport 解析与查杀信号，帮助研发快速建立问题画像并选择合适的分析入口。

Linux 内核内存回收链路

关注 direct reclaim、kswapd、watermark 与 memcg 触发条件，理解系统在压力下的回收与阻塞行为。

Android Bugreport 关键切面

通过 logcat、dumpsys、meminfo 与 killinfo 片段还原进程状态与异常时间窗。

查杀信号与驻留策略

解读 LMK、oom_adj、kill_reason 与后台驻留趋势，辅助定位关键查杀点。

内核事件：关注 reclaim 触发时机、stall 与 memcg 压力。
Bugreport 线索：定位时间窗、关键进程与异常 killinfo。
驻留与查杀：结合场景化测试判断后台稳定性。
工具入口：三大模块快速切换，统一任务节奏与产物输出。
